# NETPULS — Мережевий сканер та аналізатор вразливостей

Короткий опис
- NETPULS — графічний (CTk) інструмент для локального і віддаленого сканування підмереж, збору банерів, виявлення пристроїв і простого аналізу вразливостей.
- Скрипт: [NETPULS.py](NETPULS.py)

Ключові можливості
- Автовизначення локальної підмережі (через `netifaces` або fallback `/24`).
- Режими сканування: Тихе, Обережне, Стандартне, Кастомне, Агресивне, Віддалене.
- ARP-сканування (включно з опціональним Scapy), ICMP, TCP connect-порти, глибокий банер-грабінг (HTTP, FTP, SSH тощо).
- Спеціальні проби: mDNS, LLMNR, NetBIOS, UPnP, SNMP, SIP, MNDP (MikroTik), Winbox, SSL CN, DNS AXFR.
- Локальна офлайн-база OUI та база типових вразливостей/CVE для швидкої ідентифікації ризиків.
- Експорт результатів у CSV та стилізований HTML-звіт.

Вимоги та залежності
- Python 3.9+ (рекомендовано)
- Рекомендовані пакети (необов'язково, але додають функціонал):
  - `customtkinter` (GUI)
  - `netifaces` (точне авто-визначення інтерфейсу)
  - `scapy` (агресивні L2 операції)
  - `dnspython` (AXFR та reverse PTR)

Швидка інсталяція залежностей:

```bash
pip install customtkinter netifaces scapy dnspython
```

Запуск
- GUI (за замовчуванням):

```bash
python "NETPULS.py"
```

- З CLI опціями:

```bash
python "NETPULS.py" --interface "eth0" --disable-l2
```

Параметри CLI
- `--interface, -i` — назва інтерфейсу для сканування.
- `--disable-l2` — вимкнути ARP/Scapy L2 проби (корисно для віддалених запусків).
- `--no-virtual-filter` — не виключати віртуальні адаптери при автопідборі.

Короткий аналіз коду (що робить скрипт)
- Головний клас: `NetworkScannerApp` — будує GUI, керує станом, потіками та логікою сканування.
- Мультитредінг: використовуються `threading.Thread` та `ThreadPoolExecutor` для паралельних перевірок хостів та портів.
- L2/ARP: читає системний ARP-кеш та (якщо встановлено `scapy`) виконує агресивні ARP-запити.
- Порт-скан: нативний TCP connect (`socket.connect_ex`) з різними списками портів залежно від режиму.
- Банери та іменування: витягує HTTP <title>, Server, SSL CN, NetBIOS, mDNS/LLMNR, SNMP/UPnP/MNDP та інші джерела hostnames.
- Аналіз вразливостей: локальна `COMMON_VULNS` база + heuristics (SMBv1, anonymous FTP, DNS AXFR, відкритий SNMP/UPnP тощо).

Безпека, етика та легальність
- УВАГА: Скрипт виконує мережеве сканування та деякі проби, які можуть бути ворожими для мережевого обладнання.
- Не запускайте сканування чужих/публічних мереж без письмового дозволу — це може бути незаконно.
- Агресивні режими (Scapy, MNDP, AXFR, SIP probing, SNMP) вимагають прав адміністратора та можуть викликати захист (IDS/IPS) або порушувати роботу пристроїв.

Рекомендації та обмеження
- Для повного функціоналу (ARP агресивний, Scapy-пінги) запускайте з правами root/адміністратора.
- У віддаленому режимі уважно перевіряйте таймаути та повільність виконання, щоб не створювати зайвий трафік.
- DNS AXFR, SIP та деякі UDP проби залежать від `dnspython` і можуть бути повільні у великих мережах.
- Скрипт не проводить експлуатацію вразливостей — він дає підказки; будь-яке використання експлуатаційних кроків має відбуватись лише у рамках тестування/пентесту з дозволом.
